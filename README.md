# ClauseLens — Contract Clause Risk Analyser

An AI-powered legal tech tool that classifies contract clauses, identifies risks, and suggests revised drafts — built with Next.js and Google Gemini.

---

## Project Structure

```
clauselens/
├── data/
│   └── knowledgeBase.json        # Clause types, red flags, market standards
├── lib/
│   ├── gemini.js                 # Gemini API caller
│   ├── buildPrompt.js            # Prompt constructor
│   └── parseResponse.js         # Parses Gemini markdown → structured object
├── components/
│   ├── RiskBadge.jsx             # Colour-coded Low/Moderate/High badge
│   ├── AnalysisSection.jsx       # Reusable result panel
│   ├── LoadingSpinner.jsx        # Animated loading state
│   └── SampleClauses.jsx        # One-click example clauses
├── pages/
│   ├── _app.jsx                  # Global CSS loader
│   ├── index.jsx                 # Main UI page
│   └── api/
│       └── analyze.js            # POST /api/analyze — core API route
├── styles/
│   └── globals.css               # Tailwind directives
├── .env.local                    # API key (never commit this)
├── .gitignore
├── next.config.js                # (auto-generated by Next.js)
├── package.json
├── postcss.config.js
├── tailwind.config.js
└── vercel.json
```

## Features

| Feature | Details |
|---|---|
| Clause Classification | Identifies Indemnity, Limitation of Liability, Termination, IP, NDA, Force Majeure, Dispute Resolution |

## Setup Instructions

### 1. Get a Free Gemini API Key

1. Go to [https://aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey)
2. Sign in with a Google account
3. Click **"Create API Key"**
4. Copy the key — it looks like `AIzaSy...`

> The free tier includes generous quota — more than enough for personal use and development.

---

### 2. Clone or Download the Project

```bash
# If using git
git clone https://github.com/YOUR_USERNAME/clauselens.git
cd clauselens

# Or just download the ZIP and unzip it, then:
cd clauselens
```

---

### 3. Install Dependencies

```bash
npm install
```

This installs Next.js, React, Tailwind CSS, and PostCSS.

---

### 4. Add Your API Key

Open the file `.env.local` in the project root and replace the placeholder:

```env
GEMINI_API_KEY=AIzaSyYOUR_ACTUAL_KEY_HERE
```

> ⚠ Never commit `.env.local` to Git. It's already in `.gitignore`.

---

### 5. Run Locally

```bash
npm run dev
```

Open your browser at [http://localhost:3000](http://localhost:3000).

You should see the ClauseLens UI. Click a sample clause button or paste your own and click **"Analyse Clause"**.

## Deploying to Vercel (Free)

Vercel is the easiest way to deploy a Next.js app. It's free for personal projects.

### Step 1 — Push to GitHub

```bash
git init
git add .
git commit -m "Initial commit — ClauseLens"
git remote add origin https://github.com/YOUR_USERNAME/clauselens.git
git push -u origin main
```

### Step 2 — Connect to Vercel

1. Go to [https://vercel.com](https://vercel.com) and sign in with GitHub
2. Click **"Add New Project"**
3. Select your `clauselens` repository
4. Click **"Deploy"** — Vercel auto-detects Next.js

### Step 3 — Add the Environment Variable

Before or after deploying:
1. In your Vercel project, go to **Settings → Environment Variables**
2. Add:
   - **Name:** `GEMINI_API_KEY`
   - **Value:** `AIzaSyYOUR_ACTUAL_KEY_HERE`
   - **Environment:** Production, Preview, Development (select all)
3. Click **Save** and then **Redeploy**

Your app is now live at `https://clauselens-YOUR_ID.vercel.app`.

## How the App Works

```
User pastes clause
        ↓
pages/index.jsx  →  POST /api/analyze
        ↓
pages/api/analyze.js
   - Validates input
   - Loads knowledgeBase.json
   - Calls lib/buildPrompt.js → constructs structured legal prompt
   - Calls lib/gemini.js → hits Gemini API
   - Calls lib/parseResponse.js → extracts sections from markdown
        ↓
Returns: { clauseType, riskLevel, riskBadge, marketComparison,
           keyIssues, recommendedRevisions, revisedClause }
        ↓
pages/index.jsx renders results using:
   - <RiskBadge>          → colour-coded risk level
   - <AnalysisSection>    → each analysis panel
```

## Extending the App

### Add More Clause Types to the Knowledge Base

Edit `data/knowledgeBase.json` and add a new entry under `clauseTypes`:

```json
"paymentTerms": {
  "name": "Payment Terms Clause",
  "standardFeatures": ["Net 30 payment", "Late payment interest"],
  "redFlags": ["Payment on demand", "No interest on late payment"]
}
```

### Change the LLM Model

In `lib/gemini.js`, update the model string:

```js
// Current
const endpoint = `...models/gemini-2.0-flash:generateContent...`

// Switch to more powerful model (uses more quota)
const endpoint = `...models/gemini-1.5-pro:generateContent...`
```

### Add Export to PDF

Install `jspdf` and add a "Download Report" button in `pages/index.jsx` that calls `jspdf` with the analysis sections.

## Common Issues

| Problem | Fix |
|---|---|
| `GEMINI_API_KEY is not set` | Make sure `.env.local` exists and has your key. Restart `npm run dev` after editing. |
| `Gemini API error 400` | Your prompt may be too long. Try a shorter clause. |
| `Gemini API error 429` | Rate limit hit on free tier. Wait 60 seconds and retry. |
| Analysis sections are empty | The model returned unexpected formatting. Try the clause again — sometimes Gemini varies its format. |
| Styles not loading | Run `npm install` to ensure Tailwind is installed. |
